Designed and built a mechatronic system that integrate hardware, embedded control, and software under real-world constraints.

  • Duration: ~1.5-week focused engineering sprint

Key Features

  • Non-blocking finite state machine control

  • Modular, fully documented system for handoff

Abstract

This project involved the design and implementation of an automated control system intended to support photometric stereo scanning under real-world time constraints. Over a focused 1.5-week engineering sprint, I developed a modular mechatronic architecture integrating stepper motor–driven camera motion, synchronized LED lighting, camera triggering, and non-blocking embedded firmware. The system emphasizes deterministic timing, safe power switching, and a clear separation between hardware-specific code and core control logic to enable extensibility and reliable handoff. While the full imaging pipeline was outside the available timeline, the resulting system provides a validated foundation for continued development and has been proposed for continuation as a senior capstone project.

Project Overview

Designed and implemented an automated photometric stereo scanning system that integrates precise motion control with synchronized lighting and image capture.

Problem & Motivation

Photometric stereo requires multiple images under varied lighting to reconstruct 3D surface normals. Existing outsourced processes were slow, costly, and inconsistent, motivating the development of a modular, automated camera gantry to accelerate scans while maintaining accuracy.

The challenge was to automate this process while maintaining precise synchronization between:

  • LED lighting state changes
  • Camera triggering
  • Camera gantry motion

System Architecture

The system was designed for modularity and extensibility, allowing precise coordination of lighting and camera control while enabling future expansion or handoff to another engineer. Hardware and software responsibilities were clearly separated to simplify integration and troubleshooting.

Hardware Highlights

  • MOSFET circuit for isolated 12V LED control, ensuring consistent lighting across captures
  • Arduino Uno as the central controller, coordinating all subsystems via a structured firmware interface
  • NEMA 17 Stepper for camera gantry motion components for reproducible camera positioning
  • Relay module for camera triggering, providing isolated and reliable shutter control

Firmware Features

  • Modular finite state machine architecture coordinating camera motion, LED lighting states, and camera triggering with deterministic timing
  • Non-blocking control loops to manage concurrent tasks without timing conflicts
  • Abstracted firmware interfaces that separate control logic from hardware-specific I/O, enabling future upgrades or subsystem replacements

Subsystem Validation & Incremental Testing

Due to the compressed timeline, each subsystem was validated independently before full system integration. Mock loads, indicator LEDs, and staged wiring were used to verify firmware behavior, electrical safety, and timing assumptions prior to connecting final hardware.

LED Control Prototyping (MOSFET Switching)

Relay-based camera triggering was tested independently to validate timing, signal isolation, and repeatable shutter activation before integration with motion and lighting sequences.

LED Control Prototyping (MOSFET Switching)

High-power LED control was prototyped using MOSFET switching and indicator LEDs to verify safe, efficient power handling and firmware-controlled lighting transitions.

Stepper Motor Driver Integration

Stepper motor control was validated with microstepping and controlled motion tests to ensure deterministic positioning and reliable synchronization with lighting and camera triggers.
